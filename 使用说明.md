# imgmatch 使用说明

这个工具的作用就一个：**在屏幕上找图片，告诉你它在哪。**

---

## 你只需要记住一条命令

```
imgmatch.exe --live --template 你的图片.png
```

它会自动截屏，然后找你要的图片在屏幕上的位置。

---

## 第一步：准备一张小图片

用截图工具（Win+Shift+S）把你要找的按钮或图标截下来，保存成 PNG 图片。

比如：你想找屏幕上的"关注"按钮，就先截一张"关注"按钮的图片，存到 `imagetemplates/` 目录里。

## 第二步：运行命令

打开命令行（CMD），输入：

```
imgmatch.exe --live --template imagetemplates/关注按钮.png
```

## 第三步：看结果

**找到了：**
```
x=787, y=263, scale=125%, resolution=1920x1080
```
意思是：图片中心在屏幕坐标 (787, 263) 的位置。

**没找到：**
不会输出坐标，会报错提示。

---

## 匹配不到怎么办？

**方法 1：降低匹配门槛**（最常用）

默认门槛是 0.75，找不到就降低到 0.5 试试：
```
imgmatch.exe --live --template 你的图片.png --threshold 0.5
```

**方法 2：检查截图**

工具会自动把截到的屏幕保存到 `testdata/` 目录，打开看看目标是否在截图中。

**方法 3：权限问题**

如果截屏失败，右键 CMD → 以管理员身份运行。

---

## 两个模式的区别

| 模式 | 命令 | 什么时候用 |
|------|------|-----------|
| **Live 模式** | `--live --template 图片.png` | 自动截屏并查找（**常用**） |
| **文件模式** | `--source 截图.png --template 图片.png` | 在已有的截图文件中查找 |

99% 的情况用 Live 模式就够了。

---

## 注意事项

- 屏幕分辨率建议 1920x1080
- 截的小图片要清晰，不要模糊
- 目标窗口要在屏幕最前面，不能被遮挡
- 无需安装 Python，双击 imgmatch.exe 无法运行，必须在命令行中使用
